Motivație

De ce am construit un procesor?

Întrebări fundamentale

  • Cum transformă UC o instrucțiune binară în semnale electronice?
  • Cum comunică CPU cu periferice (tastatură, display, timer)?
  • Cum tratează hardware-ul evenimente asincrone fără să blocheze?
  • Cum extinzi un CPU general cu un accelerator dedicat?

Gap-ul identificat

  • Procesoare reale (x86, ARM) — prea complexe pentru pedagogie
  • Simulatoare clasice (LMC, MARIE) — prea simple, fără I/O real

IAS-Overclockers ocupă exact acest spațiu:
destul de simplu să fie înțeles complet,
destul de complet să demonstreze un sistem real.

Caracteristici cheie

  • 16-bit Von Neumann, memorie unificată
  • 1024 × 16b memorie, adresare 10-bit
  • 8 registre: A, X, Y, PC, SP, AR, DR, IR
  • FLAGS: Z, N, C, O
  • FSM cu 98 stări în cu.v
  • Routing AR[10]: RAM ↔ I/O

Module Verilog

cu.v alu.v memory.v io_controller.v interrupt_controller.v mining_core.v + 14 aux

Format instrucțiune (16 biți)

[15:10] opcode (6b) + [9:0] operand (10b)

Instrucțiuni unice față de cele clasice

IN port OUT port EI DI IRET WAIT MINE

Absente din orice simulator educațional existent.

WAIT — CPU idle până la prima întrerupere.
Analogul ARM WFI / RISC-V WFI. Zero cicluri irosite.

I/O

I/O cu mapare în memorie (MMIO)

Port Nume R/W Descriere

Pattern calcul eterogen modern

ARM + DMA → DMA completion IRQ
CPU + GPU → fence interrupt
CPU + NPU → ML completion IRQ
IAS-Overclockers → MINE + mining_irq

Primul simulator educațional care demonstrează acest pattern.
